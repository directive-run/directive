How It Works (Directive Flow)

  • Facts – Input text, chat history, processing state, guardrail results, allowed topics
  • Derivations – Message count, blocked/allowed count, block rate, canSend (auto-tracked)
  • Events – send (runs guardrails), clear (reset), setInput, toggleTopic
  • Constraints – offTopicDetected (priority 100) → BLOCK_MESSAGE, onTopicConfirmed (90) → ALLOW_MESSAGE
  • Resolvers – blockMessage (reject + badge), allowMessage (mock agent response)
  • Effects – Guardrail audit log updated on every result change

Input guardrails intercept messages BEFORE they reach the agent. The constraint→resolver flow handles the outcome.
