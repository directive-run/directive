Notification Stack

How It Works (Directive Flow)

  • Facts – queue (Notification[]), maxVisible, now (tick-driven), idCounter
  • Derivations – visibleNotifications (first N), hasNotifications, oldestExpired (auto-tracked)
  • Constraints – autoDismiss (priority 50, fires when oldest TTL exceeded), overflow (priority 60, trims when queue > maxVisible + 5)
  • Resolvers – dismiss — removes notification by ID from the queue
  • Events – addNotification (TTL per level: info=4s, success=3s, warning=6s, error=10s), dismissNotification, tick, setMaxVisible
  • Multi-Module – app module tracks action log; main.ts dispatches to both modules on button click

Constraints handle auto-dismiss timing and overflow — no setTimeout needed. The tick event drives time-based constraint evaluation via tickMs: 1000.
