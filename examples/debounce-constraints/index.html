How It Works (Directive Flow)

  • Events – Every keystroke updates query + queryChangedAt instantly
  • Clock – A 100ms interval ticks now, driving reactive time-based logic
  • Constraint (debounceSettled) – Fires when now - queryChangedAt ≥ debounceDelay
  • Resolver (settleDebounce) – Copies query to debouncedQuery
  • Constraint (needsSearch) – Fires when debouncedQuery settles and differs from last search
  • Resolver (search) – Calls mock API with key-based deduplication and stale result prevention

All debounce logic lives in Directive's constraint system — no timers in effects, no manual cleanup. The ticking now fact makes time a first-class reactive dependency.
