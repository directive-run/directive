How It Works (Directive Flow)

  • auth module – Facts: role, userName, token. Derivation: isAuthenticated. Events: login, logout.
  • permissions module – Cross-module constraint watches auth.isAuthenticated, fetches role-based permissions. Derivations: canEdit, canPublish, canDelete, canManageUsers, isAdmin (composition).
  • content module – Cross-module constraints watch auth and permissions. Loads articles when authenticated, gates publish/delete actions on permission derivations.

Three modules with crossModuleDeps create a reactive permission chain: login → fetch permissions → gate content actions. No manual orchestration needed.
