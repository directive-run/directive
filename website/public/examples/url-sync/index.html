How It Works (Directive Flow)

  • URL Module Facts – search, category, sortBy, page – initialized from URL params, synced bidirectionally
  • URL Effects – urlToState (popstate listener), stateToUrl (history.replaceState on fact change)
  • Products Module – Cross-module constraint reads url facts, fires FETCH_PRODUCTS when filters change
  • Resolver – fetchProducts filters/sorts/paginates mock data with 300ms simulated delay
  • Derivations – totalPages, currentPageDisplay (auto-tracked from products + url facts)

The URL module declares what the filters are. The products module uses cross-module constraints to react to changes. Effects handle the browser history sync.
