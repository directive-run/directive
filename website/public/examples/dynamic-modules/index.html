How It Works (Directive Flow)

  • createSystem – starts with only the dashboard module (multi-module mode)
  • registerModule – dynamically adds modules at runtime with full constraint/resolver integration
  • subscribeModule – subscribes to all fact changes in a specific namespace
  • Namespaced access – system.facts.counter.count, system.events.counter.increment()
  • Independent lifecycles – each module has its own constraints, resolvers, and derivations

Modules are permanent once loaded — no unregisterModule. The Reset button recreates the entire system from scratch.
