How It Works (Directive Flow)

  • Facts – User ID, 3 resource states (profile/preferences/permissions), configurable delays & failure rates, event log
  • Derivations – loadedCount, allLoaded, anyError, anyLoading, combinedStatus, canStart (auto-tracked)
  • Constraints – needsProfile (100), needsPreferences (90), needsPermissions (80) – fire concurrently when userId is set
  • Resolvers – fetchProfile (retry 3x), fetchPreferences (retry 2x), fetchPermissions (retry 3x) – exponential backoff
  • Effects – Event timeline updated on every resource status change

Constraints declare what's needed, resolvers handle the async work with retry. Derivations combine status automatically.
