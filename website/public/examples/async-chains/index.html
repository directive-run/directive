How It Works (Directive Async Chains)

  • 3 Modules – auth, permissions, dashboard — each with their own facts, derivations, constraints, and resolvers
  • Constraint Chaining – after: ["auth::validateSession"] ensures permissions only load after auth resolves
  • Cross-Module Deps – crossModuleDeps gives type-safe access to other modules' facts and derivations
  • Retry Policies – each resolver has exponential backoff (2 attempts) — use the fail rate sliders to test
  • Automatic Resolution – setting a token triggers the entire chain: validate → load permissions → load dashboard

The after ordering creates a dependency chain. Each step's constraint only evaluates once its predecessor's resolver has completed successfully.
