How It Works (Directive Flow)

  • Facts – experiments, assignments, exposures, userId, paused
  • Derivations – activeExperiments, assignedCount, exposedCount (all auto-tracked)
  • Constraints – needsAssignment (100), needsExposure (50) — constraint chain fires automatically
  • Resolvers – assignVariant (deterministic hash), trackExposure (timestamps)
  • Events – registerExperiment, assignVariant, recordExposure, pauseAll, resumeAll, reset

When an experiment is registered, the needsAssignment constraint fires, triggering the resolver to assign a variant via deterministic hash. Then needsExposure fires to track the exposure timestamp. This two-step constraint chain runs fully automatically.
