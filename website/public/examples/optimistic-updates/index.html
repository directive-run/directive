How It Works (Directive Flow)

  • Events – Apply optimistic mutations instantly (toggle, delete, add) and push undo data to a sync queue
  • Derivations – totalCount, doneCount, pendingCount, canAdd, isSyncing (all auto-tracked)
  • Constraints – needsSync (100) fires when syncQueue is non-empty and no sync is in progress
  • Resolvers – syncTodo calls mockServerSync, rolls back items on failure using stored undo data
  • Key deduplication – resolver key prevents duplicate syncs for the same operation

Optimistic mutations happen in event handlers for instant UI. The sync queue stores undo data so the resolver can roll back on failure without needing context.snapshot().
