How It Works (Directive Flow)

  • wizard module – Facts: currentStep, email, password, name, company, plan, newsletter, submitted. Derivations: step0/1/2Valid, currentStepValid, canAdvance, progress, isLastStep.
  • validation module – Cross-module constraint watches wizard.email, triggers async availability check via CHECK_EMAIL requirement.
  • Constraints – submit (60) fires on last step, advance (50) fires on other steps — both gated by currentStepValid.
  • Resolvers – advanceStep increments step, submitForm simulates 800ms API call. checkEmail simulates 500ms availability lookup.
  • Persistence – Draft data (email, name, company, plan, currentStep) persisted to localStorage via persistencePlugin.

The crossModuleDeps on the validation module gives type-safe access to facts.wizard.email for the email availability constraint.
