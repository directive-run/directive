# Directive Logo Design Concepts

Pre-launch logo design for **Directive** ([directive.run](https://directive.run)), a constraint-driven runtime for TypeScript.

Each concept includes ready-to-paste prompts for **Midjourney** and **Ideogram AI**. Preview SVG implementations at `/branding/logos`.

A right-pointing chevron (`--brand-primary`) + vertical constraint bar (`--brand-accent`) on dark slate (#0f172a). Clean but generic — reads as `> |` which overlaps with Unix operator territory.

## Concept #1: The Resolver (Convergence Point)

Three angled lines converge toward a single luminous point. Lines = constraints, convergence = resolution (system settled). Each line is a different shade of blue-to-indigo.

**Why it won:** Casey (CMO) — "High memorability, unusual in dev tools, story-ready." Sam-Tech (CTO) — "Maps precisely to reconciliation loop converging to settlement." Blake (UX) — "Clean design system extension: three-line motif for dividers, loaders, headers." Taylor (Product) — "Communicates 'things coming together' which is exactly the DX promise."

**Risks:** Dana (Frontend) — "Three lines at 16px need stroke width calibration. Favicon may reduce to 2 lines + dot." River (Streaming) — "Most dynamic of all concepts, animates beautifully."

### Midjourney Prompt
```
Minimalist geometric logomark for a developer tool called "Directive", three parallel angled lines converging to a single bright point on the right side, lines in gradient from indigo (#6366f1) to sky blue (#0ea5e9), convergence point is a luminous sky blue dot, dark slate background (#0f172a), vector style, modern developer tool brand, clean precise geometry, no text, suggests resolution and direction --ar 1:1 --s 50 --style raw
```

## Concept #8: The Lattice (Constraint Network) — NEW

Four nodes at corners of a diamond connected by faint grid lines, with a highlighted diagonal path and a bright white node at center. Represents the constraint dependency graph — edges are constraints, the highlighted path is the resolution, and the center node is settlement.

**Strengths:** Unique in dev tool space. Maps directly to constraint graph internals. Works well animated (path lights up as constraints resolve). Scalable — works as a 4-node simple version or a denser grid for marketing.

**Risks:** At 16px, nodes may blur together. Needs simplified 3-node favicon variant. Grid lines could read as "spreadsheet" without careful weight calibration.

## Logo Requirements

- Must work at 16x16 favicon (simplified version acceptable)
- Must work monochrome (single color on dark or light)
- SVG-first, PNG exports at 16/32/64/128/256/512
- WCAG contrast compliance for UI usage
- Dual lockup: logomark (icon only) + logotype (icon + "directive" in Lexend)
- Dark and light background variants

## Generation Workflow

1. **Generate top 3** — Paste prompts into Midjourney or Ideogram
2. **Iterate** — Color tweaks, proportions, stroke weight on strongest candidate
3. **SVG conversion** — Trace/recreate in Figma or code
4. **Favicon test** — Verify at 16px, 32px, 64px
5. **Update components** — `Logo.tsx`, `DirectiveLogomark.tsx`, `icon.svg`
6. **Update brand** — `brand-presets.ts` (if logo references brand colors dynamically)
7. **Update OG** — `api/og/route.tsx` OG image with new logomark
